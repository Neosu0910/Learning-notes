# 每日名詞筆記工作流程

## 觸發方式
當用戶說「新增 MMDD 筆記」（例如「新增 0508 筆記」），執行以下流程：

1. 讀取 `05note.md`（或對應月份的速記檔案），找到該日期區塊（例如 `0508`）下的所有名詞
2. 針對每個名詞撰寫詳細解釋，並 append 到 `notes/05-glossary.md`

## 檔案結構
- **速記檔**：`05note.md` — 用戶每天快速記下聽到的名詞，格式為日期標題 + 每行一個名詞
- **解釋檔**：`notes/05-glossary.md` — 展開的詳細解釋，按日期分區塊 append

## 解釋內容規範
每個名詞的解釋必須包含：
1. **定義**：清楚說明這個名詞是什麼
2. **背景/原理**：為什麼存在、解決什麼問題
3. **實際例子**：具體的使用情境或範例
4. **實作方式**（如果適用）：程式碼範例、指令、或操作步驟
5. **與相關概念的關聯**（如果適用）：和其他名詞的關係

## 格式範例
```markdown
## MMDD

**名詞名稱**
[定義與背景說明]

[原理或運作方式]

**例子：**
[具體情境說明]

**實作：**
\`\`\`語言
// 程式碼範例
\`\`\`

---
```

## 注意事項
- 內容要詳細，不要只有一兩句話
- 有程式碼的概念一定要附上可執行的範例
- 有 AWS 相關的名詞要結合實際架構情境說明
- 每個名詞之間用 `---` 分隔
